## 🎯 v5.1核心目标 (来自CORE_DATA_STRUCTURES.md)

### 架构目标
- ✅ **极简设计**: 只有Transaction和Block两种类型
- ✅ **无连接块**: 完全移除Link blocks
- ⏳ **Transaction独立**: 独立广播和存储 (Transaction.java已创建)
- ⏳ **Block轻量引用**: 只存Link (Link.java已创建)

### 性能目标
- **TPS**: 23,200 (96.7% Visa水平)
- **Block大小**: 48MB软限制
- **Max Links**: 1,485,000个/block
- **确认时间**: 6-13分钟 (Level 6)

### 核心参数
```java
// Block限制
MAX_BLOCK_SIZE = 48 * 1024 * 1024;  // 48MB
MAX_BLOCK_LINKS = 16;                // Block引用限制
MAX_TX_LINKS = 1_485_000;            // Transaction引用(block大小决定)

// Transaction data
MAX_DATA_LENGTH = 1024;              // 1KB
BASE_DATA_LENGTH = 256;              // 256字节基准

// 孤块防御
MAX_ORPHANS_PER_EPOCH = 10;          // 每epoch最多10个孤块
MAX_ROLLBACK_DEPTH = 12;             // 12 epochs回滚深度
```

## ⚠️ 重要设计决策

### 1. Transaction vs Block
- **Transaction**: 有签名(v/r/s), 无timestamp, 无PoW
- **Block**: 有PoW(nonce), 有timestamp, 无签名
- **Block引用Transaction**: 通过Link (只存hash)

### 2. Link类型
- `type=0`: 指向Transaction
- `type=1`: 指向Block

### 3. 账户模型
- 类似Ethereum: from/to/nonce
- Nonce防重放和保证顺序
- 无UTXO概念
