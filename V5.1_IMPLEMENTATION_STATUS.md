## ⚠️ 重要设计决策

### 1. Transaction vs Block
- **Transaction**: 有签名(v/r/s), 无timestamp, 无PoW
- **Block**: 有PoW(nonce), 有timestamp, 无签名
- **Block引用Transaction**: 通过Link (只存hash)

### 2. Link类型
- `type=0`: 指向Transaction
- `type=1`: 指向Block

### 3. 账户模型
- 类似Ethereum: from/to/nonce
- Nonce防重放和保证顺序
- 无UTXO概念

### 决策 1: 为何不立即激活 BlockV5？

**问题**: BlockV5 使用 Link (无 amount)，但现有代码依赖 Address.amount

**分析**:
```java
// BlockV5 设计
class Link {
    Bytes32 hash;   // 只有 hash
    LinkType type;  // TRANSACTION 或 BLOCK
    // 没有 amount！
}

// 现有代码依赖
if (ref.getAmount().subtract(MIN_GAS).isNegative()) {
    return ImportResult.INVALID_BLOCK;  // 需要 amount！
}
```

**影响范围**:
- 54+ 处 Address 引用
- BlockchainImpl.tryToConnect() - 验证逻辑
- BlockchainImpl.applyBlock() - 金额处理
