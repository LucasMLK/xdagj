## 🎯 v5.1 核心数据结构（最终版本）

### 1. BlockV5 (现在叫Block)

```java
/**
 * Block - 候选块（极简设计）
 *
 * v5.1设计原则:
 * 1. 只有Transaction和Block两种类型
 * 2. 无LinkBlock（连接块已废弃）
 * 3. Block只引用hash（轻量）
 * 4. hash缓存字段（延迟计算）
 */
@Value
@Builder(toBuilder = true)
public class Block {
    // 块头（参与hash计算）
    BlockHeader header;

    // 块体（参与hash计算）
    BlockBody body;
}

@Value
@Builder(toBuilder = true)
public class BlockHeader {
    // 基本信息（72 bytes固定）
    long timestamp;           // 时间戳（秒），epoch = timestamp / 64
    UInt256 difficulty;       // PoW难度目标值（32 bytes）
    Bytes32 nonce;            // PoW nonce（32 bytes）

    // 矿工地址（32 bytes）
    Bytes32 coinbase;         // 矿工地址

    // Hash缓存（不参与序列化）
    @Nullable
    Bytes32 hash;             // Block hash缓存（延迟计算）
}

@Value
@Builder(toBuilder = true)
public class BlockBody {
    // DAG链接列表（最多100万+）
    List<Link> links;         // 引用Transactions和其他Blocks
}
```

**关键点**:
- ✅ 所有Block都有nonce（都是候选块）
- ✅ 所有Block都有coinbase（都竞争奖励）
- ✅ epoch从timestamp推导，无需存储
- ✅ **hash是缓存字段**：不参与序列化，首次使用时计算
- ✅ **prevMainBlock不存储**：通过getBlockByHeight(height-1)查询（DRY原则）

### 2. Link (DAG边) - 33字节

```java
/**
 * Link - 纯DAG边（轻量引用）
 *
 * 设计说明:
 * - 添加type字段避免数据库查询
 * - 区分引用的是Transaction还是Block
 * - 每个link只占33字节
 */
@Value
@Builder(toBuilder = true)
public class Link {
    Bytes32 targetHash;       // 指向的块的hash（32 bytes）
    LinkType type;            // 0=Transaction, 1=Block（1 byte）
}

public enum LinkType {
    TRANSACTION(0),
    BLOCK(1);

    private final byte value;
}
```

**关键点**:
- ✅ 只有hash和type（极简）
- ✅ 无amount字段（金额在Transaction中）
- ✅ 无INPUT/OUTPUT区分（统一为引用）
- ✅ 48MB Block可引用约1,485,000个links

### 3. Transaction - 账户模型

```java
/**
 * Transaction - 账户模型交易（EVM兼容）
 *
 * 设计特点:
 * 1. 独立存在（不在Block内部）
 * 2. 独立广播和存储
 * 3. EVM兼容签名（v/r/s）
 * 4. hash缓存字段
 */
@Value
@Builder(toBuilder = true)
public class Transaction {
    // 交易数据（参与hash计算）
    Bytes32 from;             // 源账户地址（32 bytes）
    Bytes32 to;               // 目标账户地址（32 bytes）
    long amount;              // 转账金额（8 bytes）
    long nonce;               // 账户nonce（防重放、保证顺序）
    long fee;                 // 交易费用（8 bytes）
    @Nullable
    Bytes data;               // 交易数据（最多1KB）

    // 签名信息（不参与hash计算，EVM兼容）
    int v;                    // 恢复ID
    Bytes32 r;                // 签名r值（32 bytes）
    Bytes32 s;                // 签名s值（32 bytes）

    // Hash缓存（不参与序列化）
    @Nullable
    Bytes32 hash;             // Transaction hash缓存（延迟计算）
}
```

**Hash计算**:
```java
tx_hash = Keccak256(from + to + amount + nonce + fee + data)
```

**关键特性**:
- ✅ EVM兼容：ECDSA签名（secp256k1曲线），v/r/s格式
- ✅ 账户模型：类似ETH，from/to直接指定账户，nonce防重放
- ✅ 独立块类型：有自己的hash、签名，独立存储和广播
- ✅ 无timestamp：与ETH一致，nonce已足够保证顺序
- ✅ **hash缓存**：延迟计算，首次调用getHash()时计算并缓存

### 4. BlockInfo - 极简元信息（4字段）

```java
/**
 * BlockInfo - 块的元信息（极简设计）
 *
 * v5.1设计原则:
 * 1. 职责单一：只包含Block的元信息
 * 2. 极简设计：只有4个必需字段
 * 3. DRY原则：所有可推导的数据都不存储
 */
@Value
@Builder(toBuilder = true)
public class BlockInfo {
    // 核心4字段
    Bytes32 hash;             // 区块哈希（唯一标识）
    long height;              // 主链高度（height > 0 = 主块，= 0 = 孤块）
    UInt256 difficulty;       // PoW难度（32 bytes）
    long timestamp;           // 时间戳（秒）
}
```

**DRY原则** - 以下字段不存储：
- ❌ prevMainBlock → 通过 `getBlockByHeight(height - 1)` 查询
- ❌ amount/fee → 从Block的Transactions实时计算
- ❌ ref → 删除（v5.1简化）
- ❌ maxDiffLink → 删除（v5.1简化）
- ❌ snapshot → 独立的SnapshotManager管理

**关键点**:
- ✅ height字段支持快速索引和主链追溯
- ✅ height > 0 表示主块，= 0 表示孤块（极简判断）
- ✅ 高性能：最小化存储和查询开销

## 🔄 v5.1 核心设计原则

### 1. 极简架构

```
v5.1只有两种类型：
1. Transaction（独立类型，Transaction.java）
2. Block（Block.java，候选块）

❌ 删除：
- LinkBlock（连接块）
- MainBlock类（Block就是Block，通过height判断是否主块）
- TransactionBlock类（Transaction独立存在）
```

### 2. Transaction独立传播机制（关键创新）

```
XDAG的天才设计：Transaction通过P2P独立传播，与Block解耦

Step 1: Transaction创建和广播
  用户创建 tx1 并签名
  → P2P广播到全网
  → 所有节点接收并验证
  → 存入各自的mempool

Step 2: 多个候选块可以引用同一Transaction
  Epoch 100:
    节点A创建 候选块A，引用 [tx1, tx2, tx3]
    节点B创建 候选块B，引用 [tx1, tx4, tx5]  // 也引用了tx1
    所有候选块都可能引用tx1！

Step 3: 主块选择
  假设候选块B成为主块（hash最小）
  → tx1, tx4, tx5 被主块引用 ✅ 确认
  → 候选块A成为孤块
  → 但tx2, tx3并没有丢失！仍在mempool中

Step 4: 未确认交易继续等待
  tx2, tx3仍然在mempool中
  → 继续P2P传播
  → 下一个epoch的候选块可以引用它们
```

### 3. Transaction有效性（极简）

```java
/**
 * Transaction通过"被主块直接引用"变为valid
 *
 * 无需递归处理！简单直接！
 */
public boolean isTransactionValid(Transaction tx) {
    // 查询引用此tx的所有blocks
    List<Block> referencers = getBlocksReferencingTx(tx.getHash());

    // 检查是否有任何主块直接引用
    for (Block block : referencers) {
        BlockInfo info = getBlockInfo(block.getHash());
        if (info.getHeight() > 0) {  // height > 0 表示主块
            return true;
        }
    }

    return false;  // 未被主块引用
}
```

### 4. Block引用限制（v5.1）

```java
// Block引用其他Blocks的限制
MIN_BLOCK_LINKS = 1       // 至少引用1个prevMainBlock
MAX_BLOCK_LINKS = 16      // 最多引用16个其他Blocks

// Block引用Transactions无限制
MAX_TX_LINKS = 无限制     // 最多1,485,000个（由Block大小决定）
```

**关键洞察**：
- Block只存Transaction的hash（33字节），不存完整Transaction！
- 48MB Block可以引用 1,485,000个Transactions
- TPS = 1,485,000 / 64秒 = 23,200 TPS

#### 2.1 候选块创建

```java
/**
 * 创建候选块（v5.1实现）
 *
 * 关键点:
 * 1. 引用prevMainBlock
 * 2. 引用mempool中的Transactions（最多1,485,000个）
 * 3. 设置nonce、coinbase
 */
public Block createCandidateBlock() {
    // 1. 获取当前主链最新块
    Block prevMainBlock = getBlockByHeight(xdagStats.nmain);

    // 2. 从mempool获取待确认Transactions（按fee排序）
    List<Transaction> pendingTxs = mempool.getTopTransactions(
        MAX_LINKS_PER_BLOCK - 1  // 留1个位置给prevMainBlock
    );

    // 3. 创建links
    List<Link> links = new ArrayList<>();

    // 添加prevMainBlock引用
    links.add(Link.builder()
        .targetHash(prevMainBlock.getHash())
        .type(LinkType.BLOCK)
        .build());

    // 添加Transaction引用
    for (Transaction tx : pendingTxs) {
        links.add(Link.builder()
            .targetHash(tx.getHash())
            .type(LinkType.TRANSACTION)
            .build());
    }

    // 4. 创建候选块
    long timestamp = XdagTime.getCurrentTimestamp();
    UInt256 difficulty = calculateDifficulty();
    Bytes32 coinbase = keyPair2Hash(wallet.getDefKey());

    BlockHeader header = BlockHeader.builder()
        .timestamp(timestamp)
        .difficulty(difficulty)
        .nonce(Bytes32.ZERO)  // 待PoW计算
        .coinbase(coinbase)
        .build();

    BlockBody body = BlockBody.builder()
        .links(links)
        .build();

    Block candidateBlock = Block.builder()
        .header(header)
        .body(body)
        .build();

    return candidateBlock;
}
```

#### 2.2 候选块导入和验证

```java
/**
 * 导入候选块（v5.1实现）
 *
 * 验证规则:
 * 1. PoW验证：hash <= difficulty
 * 2. Timestamp验证
 * 3. Links验证：Transaction和Block存在性
 * 4. 主块选择：hash最小的成为主块
 */
public synchronized ImportResult tryToConnect(Block block) {
    // 1. 基本验证
    if (block.getHeader().getTimestamp() > XdagTime.getCurrentTimestamp() + MAIN_CHAIN_PERIOD / 4) {
        return ImportResult.INVALID_BLOCK.setErrorInfo("Block timestamp too far in future");
    }

    // 2. PoW验证
    Bytes32 blockHash = block.getHash();
    UInt256 difficulty = block.getHeader().getDifficulty();
    if (blockHash.compareTo(difficulty.toBytes()) > 0) {
        return ImportResult.INVALID_BLOCK.setErrorInfo("PoW verification failed");
    }

    // 3. 检查是否已存在
    if (isExist(blockHash)) {
        return ImportResult.EXIST;
    }

    // 4. 验证links
    List<Link> links = block.getBody().getLinks();

    // 验证Block引用数量（1-16个）
    long blockLinks = links.stream()
        .filter(link -> link.getType() == LinkType.BLOCK)
        .count();

    if (blockLinks < MIN_BLOCK_LINKS || blockLinks > MAX_BLOCK_LINKS) {
        return ImportResult.INVALID_BLOCK.setErrorInfo("Invalid block reference count");
    }

    // 验证引用存在性
    for (Link link : links) {
        if (link.getType() == LinkType.TRANSACTION) {
            Transaction tx = transactionStore.getTransaction(link.getTargetHash());
            if (tx == null) {
                return ImportResult.NO_PARENT.setHash(link.getTargetHash());
            }
        } else {
            Block refBlock = getBlockByHash(link.getTargetHash(), false);
            if (refBlock == null) {
                return ImportResult.NO_PARENT.setHash(link.getTargetHash());
            }
        }
    }

    // 5. 初始化BlockInfo（孤块，height = 0）
    BlockInfo initialInfo = BlockInfo.builder()
        .hash(blockHash)
        .height(0L)  // 孤块
        .difficulty(difficulty)
        .timestamp(block.getHeader().getTimestamp())
        .build();

    // 6. 保存Block和BlockInfo
    blockStore.saveBlock(block);
    blockStore.saveBlockInfo(initialInfo);

    // 7. 检查是否应该成为主块
    checkIfShouldBeMainBlock(block);

    return ImportResult.IMPORTED_NOT_BEST;
}
```

#### 2.3 主块选择

```java
/**
 * 检查候选块是否应该成为主块（v5.1实现）
 *
 * 规则：同一epoch中，hash最小的成为主块
 */
private void checkIfShouldBeMainBlock(Block block) {
    long epoch = XdagTime.getEpoch(block.getHeader().getTimestamp());

    // 获取当前主块（如果存在）
    Block currentMainBlock = getMainBlockByEpoch(epoch);

    if (currentMainBlock == null) {
        // 没有主块，此block成为主块
        setAsMainBlock(block);
    } else {
        // 比较hash，选择最小的
        if (block.getHash().compareTo(currentMainBlock.getHash()) < 0) {
            // 新block hash更小，替换主块
            unsetMainBlock(currentMainBlock);
            setAsMainBlock(block);
        }
    }
}

/**
 * 设置为主块（v5.1实现）
 */
private void setAsMainBlock(Block block) {
    // 1. 计算height
    long height = xdagStats.nmain + 1;

    // 2. 更新BlockInfo
    BlockInfo newInfo = BlockInfo.builder()
        .hash(block.getHash())
        .height(height)  // height > 0 表示主块
        .difficulty(block.getHeader().getDifficulty())
        .timestamp(block.getHeader().getTimestamp())
        .build();

    blockStore.saveBlockInfo(newInfo);

    // 3. 执行Block引用的所有Transactions
    XAmount totalFees = executeBlockTransactions(block);

    // 4. 更新统计
    xdagStats.nmain = height;
    xdagStats.setMaxdifficulty(newInfo.getDifficulty().toBigInteger());

    // 5. 保存状态
    blockStore.saveXdagStatus(xdagStats);
}

/**
 * 执行Block引用的所有Transactions（v5.1实现）
 */
private XAmount executeBlockTransactions(Block block) {
    XAmount totalFees = XAmount.ZERO;

    List<Link> links = block.getBody().getLinks();

    for (Link link : links) {
        if (link.getType() == LinkType.TRANSACTION) {
            Transaction tx = transactionStore.getTransaction(link.getTargetHash());

            if (tx != null) {
                // 验证并执行Transaction
                if (validateAndExecuteTransaction(tx)) {
                    totalFees = totalFees.add(XAmount.of(tx.getFee()));

                    // 从mempool移除
                    mempool.remove(tx.getHash());
                }
            }
        }
    }

    return totalFees;
}
```

```java
/**
 * CachedBlockStore - 块存储缓存层（v5.1简化）
 *
 * 只缓存BlockInfo，不缓存Block（Block可以很大，不适合缓存）
 */
public class CachedBlockStore implements BlockStore {
    private final BlockStore delegate;

    // BlockInfo缓存（小，可以缓存）
    private final Cache<Bytes32, BlockInfo> blockInfoCache;

    // 主链索引缓存
    private final Cache<Long, Bytes32> heightCache;

    public CachedBlockStore(BlockStore delegate, long cacheSize) {
        this.delegate = delegate;
        this.blockInfoCache = CacheBuilder.newBuilder()
            .maximumSize(cacheSize)
            .expireAfterAccess(5, TimeUnit.MINUTES)
            .build();
        this.heightCache = CacheBuilder.newBuilder()
            .maximumSize(50_000)
            .build();
    }

    @Override
    public void saveBlock(Block block) {
        delegate.saveBlock(block);
    }

    @Override
    public Block getBlockByHash(Bytes32 hash, boolean isRaw) {
        // Block不缓存，直接查询
        return delegate.getBlockByHash(hash, isRaw);
    }

    @Override
    public void saveBlockInfo(BlockInfo info) {
        blockInfoCache.put(info.getHash(), info);
        delegate.saveBlockInfo(info);
    }

    @Override
    public BlockInfo getBlockInfo(Bytes32 hash) {
        return blockInfoCache.get(hash, () -> delegate.getBlockInfo(hash));
    }

    @Override
    public Bytes32 getMainBlockHashByHeight(long height) {
        return heightCache.get(height, () -> delegate.getMainBlockHashByHeight(height));
    }
}
```

#### 4.2 Mempool

```java
/**
 * Mempool - 待确认交易池（v5.1设计）
 */
public class Mempool {
    // 按fee优先级排序
    private final PriorityQueue<Transaction> txQueue;

    // Transaction索引（快速查找）
    private final Map<Bytes32, Transaction> txMap;

    /**
     * 添加Transaction到mempool
     */
    public boolean addTransaction(Transaction tx) {
        // 1. 验证签名
        if (!tx.verifySignature()) {
            return false;
        }

        // 2. 验证nonce
        long expectedNonce = addressStore.getNonce(tx.getFrom()) + 1;
        if (tx.getNonce() != expectedNonce) {
            return false;
        }

        // 3. 验证余额
        XAmount balance = addressStore.getBalance(tx.getFrom());
        XAmount required = XAmount.of(tx.getAmount() + tx.getFee());
        if (balance.compareTo(required) < 0) {
            return false;
        }

        // 4. 加入mempool
        txQueue.offer(tx);
        txMap.put(tx.getHash(), tx);

        return true;
    }

    /**
     * 获取fee最高的N个Transactions
     */
    public List<Transaction> getTopTransactions(int limit) {
        return txQueue.stream()
            .limit(limit)
            .collect(Collectors.toList());
    }
}
```

```
关键创新：
1. Transaction独立于Block存在
2. 独立P2P广播和存储
3. Block只引用Transaction的hash
4. 孤块不影响Transaction（仍在mempool中）
5. 下个epoch可以重新引用

结果：
- 即使孤块率99%，TPS也不受影响
- Transaction确认率提升
- 用户体验改善
```

### 4. Hash缓存机制

```java
// Block hash缓存
public Bytes32 getHash() {
    if (header.hash == null) {
        header.hash = Keccak256(serialize(header) + serialize(body));
    }
    return header.hash;
}

// Transaction hash缓存
public Bytes32 getHash() {
    if (this.hash == null) {
        this.hash = Keccak256(from + to + amount + nonce + fee + data);
    }
    return this.hash;
}
```
