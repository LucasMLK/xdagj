三、原因分析
================================================================================

为什么提高Pool难度50倍，拒绝率却没有明显改善？

根本原因: Miner的nonceRange配置

  nonceRange = 10000 的含义:
    • Miner每次最多尝试10000个nonce值
    • 无论是否找到符合难度的share
    • 10000次尝试后立即向Pool请求新工作
    • 这导致Miner以固定频率提交工作

  实际行为:
    • Miner每0.02秒完成10000次哈希计算
    • 每0.02秒向Pool请求新工作
    • 提交频率：50次/秒（固定）
    • 与Pool难度设置无关！

  对比正常挖矿:
    • 正常情况：找到符合难度的share才提交
    • 当前情况：时间到了就提交（无论是否找到）

================================================================================
四、区块拒绝的真正原因
================================================================================

拒绝率高不是因为Pool难度设置：

  真正原因是 "候选区块竞争":

    1. Pool基于Candidate A生成挖矿任务
    2. Miner找到多个符合区块难度的nonce
    3. Pool提交第1个 → ✅ 被接受
    4. Candidate立即更新为B
    5. Pool提交第2、3、4个... → ❌ 全部拒绝（基于旧Candidate A）

  数学模型:
    • 假设64秒内找到10个符合区块难度的nonce
    • 第1个被接受
    • 剩余9个在提交前candidate已更新
    • 拒绝率 = 9/10 = 90%

  两节点竞争:
    • Suite1产生区块 → Suite1 candidate更新
    • Suite2产生区块 → 通过P2P传播 → Suite1 candidate更新
    • 更频繁的candidate更新 → 更高的拒绝率
