## Project Overview

### What is v5.1?

v5.1 is a ground-up rewrite of XDAG's core architecture, introducing:

```
v5.1 Architecture:
├── Storage Layer     - Unified DagStore for Block/Transaction/Account
├── Cache Layer       - 13.8 MB L1 Caffeine cache (>90% hit rate)
├── Consensus Layer   - Epoch-based DAG with cumulative difficulty
├── Processing Layer  - Transaction validation and state management
└── Sync Layer        - Hybrid Main Chain + Epoch DAG protocol
```

### Why v5.1?

**Legacy Issues (v4.x)**:
- Block size limited to 512 bytes
- TPS capped at 100 (far below network needs)
- Tight coupling between components
- Limited scalability
- No smart contract support

**v5.1 Solutions**:
- 48 MB blocks with 1.485M links capacity
- 23,200 TPS target (232x improvement)
- Clean separation of concerns
- EVM-compatible account state
- Independent, testable components

## Architecture Highlights

### Design Principles

1. **Separation of Concerns**: Clear boundaries between storage, consensus, and sync
2. **Independence**: Components can be tested and evolved independently
3. **Immutability**: Thread-safe data structures prevent race conditions
4. **Performance**: L1 cache + optimized indexing for high throughput
5. **Future-Proof**: EVM compatibility for smart contracts

### Key Design Decisions

**Decision 1: Unified DagStore**
- Single source of truth for Block/Transaction persistence
- Eliminates data inconsistencies
- Simplified architecture

**Decision 2: Epoch-Based Consensus**
- 64-second epochs with smallest hash wins
- Clear and predictable
- Fair competition

**Decision 3: Immutable Block Structure**
- Thread-safe by design
- No accidental mutations
- Better for functional programming

**Decision 4: Separation of Block and BlockInfo**
- Block = serialized data (hash, header, links)
- BlockInfo = runtime metadata (height, difficulty)
- Smaller serialization size

**Decision 5: Static Block Factory Methods**
- Block creation logic in Block class
- DagChain provides convenience wrappers
- No code duplication

**Decision 6: Independent DagChain**
- Clean break from legacy Blockchain
- v5.1-specific semantics
- Easier migration path
