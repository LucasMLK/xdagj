## 5. 高度编号差异分析 ⚠️ (预期行为)

### 观察
Node1的区块高度始终比Node2高1 (Node1 = Node2 + 1)

### 根本原因

**时间线**:
- **13:45:43** - Node1启动，创建genesis
- **13:46-13:49** - Node1独立挖出3个区块（无peer连接）
- **13:49:43** - Node2启动（晚了4分钟）
- **13:50-13:52** - Node2独立挖出2个区块
- **13:52:17** - 两节点P2P连接成功，开始同步
- **13:52:17+** - 收敛到相同主链

**独立挖矿期间的区块**:

Node1独立区块（3个）:
```
Height 2: 0x5947ce4c... (diff 3)
Height 3: 0x7207d6d6... (diff 5)
Height 4: 0xb7ee06c9... (diff 6)
```

Node2独立区块（2个）:
```
Height 2: 0x4e62377d... (diff 4)
Height 3: 0xa25291ef... (diff 5)
```

**收敛点**: Block 0x75bf1f9a... (diff 7)
- Node1视角: Height 5
- Node2视角: Height 4
- 差值 = 1 (保持至今)

### 为什么这是预期行为？

1. **DAG架构特性**: 不同节点可以有不同的早期历史
2. **累积难度一致**: 真正的共识指标（完全相同 ✅）
3. **主链一致**: 从收敛点开始的所有区块都相同
4. **高度编号**: 仅是本地计数器，反映独立挖矿历史

### 是否需要修复？

**建议**: ❌ 不需要修复

**理由**:
- 这是合理的DAG实现特性
- 累积难度才是权威的链长度指标
- 高度差异不影响共识和数据一致性
- 修复需要重大架构改动，收益很小

如果未来需要统一高度，可以考虑：
- 基于累积难度而不是高度来标识区块
- 实现全网高度共识协议
- 引入检查点机制
